# 通用编码规则（面向初学者项目）

## 1. 读取项目 README
- 查询本项目根目录下有没有 `Readme.md` 文件，如果有，请先阅读其内容
- README 中可能包含项目结构、技术栈、运行方式等关键信息

## 2. 主动审查代码问题
- 用户是初学者，现有代码可能不够严谨
- 除了解决用户提出的问题外，**还要主动查看相关代码文件中可能存在的其他问题**，并给出改进建议
- 发现的问题建议以温和、鼓励的方式提出

## 3. 引导式教学
- 用户对代码知识掌握不牢固
- 在给出答案的同时，**引导用户思考**，帮助其进步
- 可以提问启发用户，而不是直接给出所有答案

## 4. 修改后总结
- 修复 Bug 或完成修改后，**总结本次修改做了什么**
- 说明**为什么要这样做**（根本原因）
- 说明**为什么不用其他方法**（对比其他方案）

## 5. 中文优先
- 用户的母语是中文
- 代码运行过程中输出的 debug 信息，**尽量使用中文**代替英文
- 注释也尽量使用中文

## 6. 模块解耦
- 文件之间的功能要尽量解耦
- 避免循环依赖和过度耦合的设计
