# C 语言编码规范

## 1. 函数必须先定义再调用
- C 语言中，函数在被调用前必须已有声明或定义，否则编译器可能报错（隐式声明在 C99 后已被移除）。
- 如果函数定义在调用点之后，**必须在调用前添加函数前向声明（forward declaration）**。
- 前向声明统一放在文件顶部（`#include` 之后）。

## 2. 编译零警告
- 编写的 C 代码**必须做到无警告编译**（`-Wall -Wextra -Wpedantic` 级别）。
- 常见需留意的地方：
  - 所有 `switch` 语句必须有 `default` 分支。
  - 未使用的参数用 `(void)param;` 或 `__attribute__((unused))` 消除警告。
  - 返回值不应被忽略（除非明确用 `(void)` 丢弃）。
  - 类型转换要显式，避免隐式符号/精度丢失。
  - 比较有符号/无符号数时注意类型匹配。
- 如果选用第三方库接口无法避免警告，用编译器 `#pragma` 局部屏蔽。

## 3. 头文件最小暴露原则
- `.h` 文件只暴露**必须被外部使用的**接口（函数声明、宏、类型定义）。
- **仅在同名 `.c` 文件内部使用的函数、全局变量、辅助宏，一律不要写在 `.h` 中**。
- 内部函数用 `static` 修饰，限制在本编译单元内。
- 如果 `.h` 中需要结构体定义，尽量使用**不透明指针（opaque pointer）** 手法，仅在 `.h` 中前向声明，定义留在 `.c` 中。
